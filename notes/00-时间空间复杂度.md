# 时间 / 空间复杂度入门（笔记）
> 原文：[时间空间复杂度入门](https://labuladong.online/zh/algo/intro/complexity-basic/)
## 总结
- **时间复杂度**：用于衡量算法的执行效率，「代码运行次数」随输入规模 \(n\) 的增长速度。
  - 简单理解：看 for 循环的「最大嵌套层数」，注意有些操作背后有隐含循环（比如初始化长度为 n 的数组），。
- **空间复杂度**：用于衡量算法的内存消耗，「额外占用的内存」随 \(n\) 的增长速度。
  - 简单理解：看申请了多少空间储存数据，输入参数本身不算在空间复杂度里。
- **只保留最高增长项，用 Big O 记号**：可以忽略常数和低阶项。
  - 表示法举例：\(O(1), O(\log n), O(n), O(n^2)\) …
  - 复杂表达式只保留最高项：\(O(2n^2 + 3n + 1) \Rightarrow O(n^2)\)。
- **考虑最坏情况的复杂度**：可以忽略常数和低阶项。

## 示例
### 示例一：遍历数组求和
```cpp
// 输入一个整数数组，返回所有元素的和
int getSum(int[] nums) {
    int sum = 0;
    for (int i = 0; i < nums.length; i++) {
        sum += nums[i];
    }
    return sum;
}
时间复杂度：(O(n))，因为有一个顺序遍历的 for 循环。
空间复杂度：(O(1))，只用了常数个变量（sum 和循环变量），nums 是输入不算。
示例二：有条件的循环
// 当 n 是 10 的倍数时，计算累加和，否则返回 -1
int sum(int n) {
    if (n % 10 != 0) {
        return -1;
    }
    int sum = 0;
    for (int i = 0; i <= n; i++) {
        sum += i;
    }
    return sum;
}
时间复杂度：最坏情况需要执行 for 循环，所以是 (O(n))。
空间复杂度：(O(1))。
我的理解：即使有时候直接返回 (O(1))，复杂度统一看「最坏情况」。

示例三：双重循环（两数之和暴力）
// 数组是否存在两个数，它们的和为 target？
boolean hasTargetSum(int[] nums, int target) {
    for (int i = 0; i < nums.length; i++) {
        for (int j = i + 1; j < nums.length; j++) {
            if (nums[i] + nums[j] == target) {
                return true;
            }
        }
    }
    return false;
}
时间复杂度：两层循环，最坏情况接近 (n \times n)，即 (O(n^2))。
空间复杂度：(O(1))，只用了常数个变量 i, j。
我的理解：即使内层循环不完全遍历、可能提前 return，依然按「最大可能次数」估算。

示例四：申请数组空间
void exampleFn(int n) {
    int[] nums = new int[n];
}
时间复杂度：初始化数组元素相当于一个隐藏的 for 循环，(O(n))。
空间复杂度：新开了长度为 (n) 的数组，(O(n))。
示例五：新数组 + 遍历
// 输入一个整数数组，返回一个新的数组，新数组的每个元素是原数组对应元素的平方
int[] squareArray(int[] nums) {
    int[] res = new int[nums.length];
    for (int i = 0; i < nums.length; i++) {
        res[i] = nums[i] * nums[i];
    }
    return res;
}
时间复杂度：初始化 res 是 (O(n))，for 遍历也是 (O(n))，合在一起还是 (O(n))。
空间复杂度：新数组 res 长度为 (n)，所以 (O(n))。
